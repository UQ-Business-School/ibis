UQ BIN Alerts

The alert components are used to notify users of important information or changes on a page attracting the user's attention without interrupting the user's task. The alerts are mainly categorised as page alerts and global alerts.

Page Alerts

Page alerts are light variants of alerts that are used locally on pages along with components or on its own to notify users of important information or changes on a page. However, the existing role alerts take on a unique role for screen readers. When a role="alert" is found on the page by a screen reader, it will stop what it is currently narrating and then say, alert followed by what is written within the tag. Additionally, you can only have one "alert" per web page, and the last div element is the one that will be read by the screen reader. This is not what we recommend to use in a normal learning module, and therefore have elected to designate them as notes instead. A role="note" is used when the content is supplementary to the material on the page, and not essential to the main topic. For example, on a module page written about statistics, we can instruct a student that a copy of the reading materials is located at a link, and a note that only a certain number of students may view it at any one point.

To get started:

Make sure that you have all of the CSS code linked into the page:

<link rel="stylesheet" href="https://uq-business-school.github.io/styleguide/css/superstyle.css" />
<script src="https://cdn.jsdelivr.net/npm/aos@2.3.4/dist/aos.js"></script>

If you want animations, then you'll also need to load in the animations CSS and Javascript too:

<!-- Put the following at the bottom of the page before closing out the body -->
<script>
AOS.init({
duration: 1200,
offset: 20,
})
</script>

For more information about how the animation properties work, see AOS github repository: https://github.com/michalsnik/aos?tab=readme-ov-file.
